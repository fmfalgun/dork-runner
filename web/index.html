DORK RUNNER

Google + DDG dork query launcher with passive OSINT APIs. 400+ built-in patterns, community dork board.

CLI: bash ddgr-dorks.sh nmap.org
→ Open Runner GitHub ↗
> WHAT ARE DORKS

Search engine operators that expose sensitive information indexed by mistake. Patterns like site:target.com filetype:env or intitle:"index of" /admin surface config files, credentials, directory listings, and admin panels that were never meant to be public.

Used in passive OSINT reconnaissance — no direct target interaction required.

> WHY THIS TOOL

Combines query generation, passive APIs (Wayback Machine CDX, GitHub Code Search, crt.sh), optional Google CSE in-page results, and a community dork board — all in one interface.

400+ built-in dork patterns across 13 categories. Drop in a domain and it builds all applicable queries automatically.

> HOW IT WORKS
User enters domain ↓ Runner (dork-runner.html) ↓ ↓ ↓ Option A — Tab Launcher Opens dork queries in new tabs Option B — Passive APIs Wayback CDX + GitHub Search + crt.sh Option C — Google CSE API In-page results (optional key required)

DDG limitation: DuckDuckGo has no public search results API. Their API endpoint only returns instant answers (definitions, calculations, knowledge panel data) — not web search results. This means DDG mode is tab launcher + passive APIs only. There is no way to get DDG web results programmatically without scraping.

Google CSE: Google's Programmable Search Engine API returns actual web search results in-page for up to 100 queries/day free (10 results per query). Requires a CSE ID and API key — see setup below.

> CSE SETUP — GOOGLE IN-PAGE RESULTS
01. Go to programmablesearchengine.google.com → New search engine
02. Under "Sites to search" select Search the entire web — this lets you dork any domain, not just your own site
03. After creation, open the engine → Overview → copy the Search engine ID (the cx value)
04. Go to console.cloud.google.com → APIs & Services → Library → search Custom Search API → Enable
05. APIs & Services → Credentials → Create credentials → API key → copy the key
06. In Runner, click ⚙ Setup → paste both values → Save. Keys are stored in localStorage only — never sent anywhere except the Google CSE API.
07. Free tier: 100 queries/day, 10 results each. Pricing ↗ if you need more.
